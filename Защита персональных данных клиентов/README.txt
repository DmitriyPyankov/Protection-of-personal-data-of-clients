Описание задания:

Необходимые данные клиентов страховой компании «Хотить потоп».

Необходимо разработать такой метод преобразования данных, чтобы было трудно восстановить личную информацию и обосновать 
корректность его работы. При этом необходимо иметь такие данные, чтобы качество моделей машинного обучения не ухудшилось 
при трансформации.

В качестве модели машинного обучения используется линейная регрессия.

Описание набора данных

Независимые признаки: пол, возраст и зарплата застрахованного, количество членов семьи;
Целевой признак: количество страховых выплат клиентам за последние 5 лет.
Цель работы — разработать метод преобразования данных, защищающий личную информацию клиентов страховой компании «Хоть потоп» 
и не влияющий на работу модели линейной регрессии.

Проведенная работа:

исследовательский анализ и предварительная обработка данных;
Построение моделей линейной регрессии и анализ анализа матричных операций над набором данных по качеству моделей;
Алгоритм преобразования датaсета для защиты клиентов данных компании;
применение и проверка алгоритма защиты данных;
Проверка корректности разработанной модели линейной регрессии с аналогичной моделью из библиотеки SKlearn.
